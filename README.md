## Part 2. Press Release (성공한 미래의 보도자료)

### 헤드라인
**문자 주문을 데이터로 바꾸다: 소상공인 주문 관리 시간을 50% 줄인 자동화 시스템 출시**

### 리드 문단
[서울, 2026년 3월 1일] 문자 주문을 주로 사용하는 소상공인의 반복적인 주문 정리 문제를 해결하는 **문자 주문 자동 수집·정형화 시스템**이 성공적으로 출시되었다. 본 시스템은 문자로 들어오는 주문, 입금 완료 캡처 이미지, 배송 요청 사항을 자동으로 수집·정리하여 주문 처리 시간을 평균 50% 이상 단축하는 성과를 달성했다.

### 해결한 문제
- **문제 1**: 문자 주문 형식이 제각각이라 수작업 정리에 평균 주문당 **약 5분** 소요
- **문제 2**: 주문 오입력·누락 시 오배송, 고객 불만 및 신뢰도 하락으로 이어지며 이는 **재구매율 감소 및 매출 하락으로 직결**
- **문제 3**: 주문량 증가 시 엑셀 정산 및 관리 한계 발생

### 제공한 솔루션
- **솔루션 1**: 판매자 휴대폰 문자(SMS/MMS) 자동 수집
- **솔루션 2**: 주문·입금·배송 요구사항 자동 분리 및 정형화
- **솔루션 3**: 주문 단위 엑셀 자동 생성 및 관리

### 고객 변화 (Before / After)
- **Before**: 문자·이미지·메모를 오가며 수작업 정리
- **After**: 주문 1건 = 데이터 1행, 입금·배송 정보까지 한 번에 확인

**핵심 가치**: 주문 정리에 쓰이던 시간을 판매와 고객 응대에 재투자

### 성과 지표
- 주문 정리 시간: 평균 **5분** → 1.5분 (**70% 감소**)
- 주문 오류·누락률: 5% → 1% 이하 (**80% 이상 개선**, 고객 신뢰도 유지)
- 엑셀 정산 준비 시간: 일 30분 → 5분

### 고객 후기 (가상)
“문자 주문이 많아질수록 엑셀이 지옥이었는데, 이제는 자동으로 정리됩니다.” – 온라인 판매자 A

### 오너십 코멘트

## Part 3. 외부 고객용 FAQ

### Q1. 이 서비스는 무엇이며 누구를 위한 건가요?
- **무엇**: 문자 주문을 자동으로 정리해주는 시스템
- **대상**: 문자로 주문을 받는 모든 판매자
- **가치**: 주문·입금·배송 정보를 한 번에 관리

### Q2. 기존 방식과 무엇이 다른가요?
- 이전: 문자 확인 → 메모 → 엑셀 입력
- 이후: 문자 수신 즉시 자동 정리
- 차별점: 입금 캡처·배송 요청까지 함께 관리

### Q3. 언제부터 사용할 수 있나요?
- 2026년 3월 MVP 공개
- 신규·기존 판매자 모두 대상

### Q4. 설치나 조건이 필요한가요?
- Android: 자동 수집 설정
- iOS: 단축어 기반 반자동

### Q5. 비용이 발생하나요?
- MVP 단계에서는 무료 또는 저비용 정책
- 수작업 시간 절감 효과로 비용 대비 효율 확보

### Q6. 개인정보는 안전한가요?
- 민감 정보(현관 비밀번호 등)는 분리·권한 제한 저장

## Part 6. 성과 측정, 수익성 및 확산 계획

### 1) 가격 정책 및 고객 가치 (주문량 기반 구독)

본 서비스는 판매자마다 주문량 편차가 크다는 현실을 반영해 **주문 건수 기반 구독 요금제**를 적용한다. 이는 소규모 판매자의 진입 장벽을 낮추는 동시에, 대량 주문 판매자에게도 합리적인 비용 구조를 제공하기 위함이다.

#### 주문량 기준 요금 구간 (권장안)

| 구분 | 월 주문 건수 | 월 구독료 | 판매자 체감 포인트 |
|---|---|---|---|
| Starter | ~300건 (일 10건 내외) | 9,900원 | 엑셀 최소화, 기본 자동 정리 |
| Standard | ~600건 (일 20건 내외) | 19,900원 | 엑셀 업무 제거, 입금·배송 통합 |
| Pro | ~1,500건 (일 50건 내외) | 39,900원 | 대량 주문 안정 처리, 신뢰도 유지 |
| Enterprise | 1,500건 이상 | 별도 협의 | 전용 모델·연동·SLA |

---

#### 판매자 관점 시간·비용 대비 효과

- 주문 1건 평균 정리 시간: 5분
- 월 300건 기준: 약 25시간
- 월 600건 기준: 약 50시간
- 월 1,500건 기준: 약 125시간

→ 구독료는 **절감되는 시간 가치의 약 3~8% 수준**으로 설계됨

### 3) 개발사 관점 수익성(단순 추정)

#### 기본 가정
- 월 구독료: 20,000원

### 4) 확장성이 높은 이유 (개발사 관점)

- 주문 데이터가 쌓일수록 모델 정확도 개선
- 판매자 추가 시 **한계 비용 거의 증가 없음**
- 확장 가능 영역
  - 카카오톡/메신저 주문 수집
  - 정산·회계 자동화
  - 배송/퀵 연동
  - 업종별 특화 모델

→ **구독형 반복 매출 + 데이터 자산 축적 구조**
