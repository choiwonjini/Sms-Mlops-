# Press Release (PR) – 문자 주문 정형화 & MLOps 기반 주문 자동화 서비스

---

## 1. 제목 (Heading)
**문자 주문을 데이터로 바꾸다: 소상공인 주문 관리의 새로운 표준**

---

## 2. 부제 (Sub-heading)
문자로 흩어지던 주문 정보를 자동으로 정리·저장·엑셀화하는 운영 중심 MLOps 솔루션.  
판매자는 주문 정리에 쓰던 시간을 줄이고, 고객은 더 빠르고 정확한 처리를 경험합니다.

---

## 3. 문제 제기 (Problem)

현재 많은 소상공인과 개인 판매자는 주문을 카카오톡이나 전용 쇼핑몰이 아닌 **휴대폰 문자(SMS)** 로 받고 있습니다. 주문 내용에는 상품명, 수량, 금액, 송금 계좌, 요청 사항이 포함되지만, 형식은 고객마다 제각각입니다.

그 결과:
- 주문 정보를 다시 적어야 하는 반복 노동
- 금액·수량 누락 또는 오입력 위험
- 주문이 늘어날수록 관리가 불가능해지는 구조

문제의 핵심은 판매자의 실수가 아니라, **비정형 문자 주문을 처리할 수 있는 시스템이 없다는 점**입니다.

---

## 4. 솔루션 (Solution)

본 서비스는 문자 주문을 자동으로 수집하고, 핵심 정보만 추출해 **정형 데이터로 저장한 뒤 엑셀로 변환**합니다.

특히 실제 현장에서는 주문자가 **입금 완료 후 송금 내역 캡처 이미지**를 문자로 함께 전송하는 경우가 많다는 점을 설계에 반영했습니다.

핵심은 단순한 AI 기능이 아니라, 실제 운영에 맞춘 **MLOps 기반 구조**입니다.

- 문자 주문 원문과 함께 **송금 완료 캡처 이미지 동시 저장**
- 상품명, 수량, 금액, 계좌 정보 자동 추출
- 주문 시 함께 전달되는 **배송 관련 기타 요구사항 자동 분리 저장**
  - 예: 현관 비밀번호, 경비실 요청, 특정 시간 배송 요청 등
- 민감 정보(현관 비밀번호 등)는 일반 주문 정보와 분리 저장 및 접근 권한 제한
- 캡처 이미지 기반 입금 여부 확인 보조 (OCR + 메타데이터)
- 신뢰도 기준으로 자동 처리와 수동 확인을 구분
- 주문 데이터, 입금 증빙, 배송 요구사항이 함께 엑셀로 관리 가능

이를 통해 판매자는 주문·입금·배송 정보를 여러 메시지를 오가며 확인할 필요 없이, 한 화면·한 파일에서 맥락 단위로 확인할 수 있으며 ‘정리 작업’에서 벗어나 ‘판매’에 집중할 수 있습니다.

# Frequently Asked Questions (FAQ)

---

## 내부(운영/개발) 관점 FAQ

### Q1. 문자 형식이 제각각인데 정말 자동 처리가 가능한가요?
완전 자동은 아닙니다. 대신 전체 주문의 약 70~90%는 자동 처리되고, 나머지는 확인 대상로 분리됩니다. 이 구조가 실제 운영에서 가장 안정적입니다.

### Q2. 모델이 틀리면 어떻게 되나요?
모든 결과는 원문 문자와 함께 저장됩니다. 오류가 발생하더라도 언제든 원인을 추적하고 모델을 개선할 수 있습니다.

### Q3. 주문 데이터가 쌓이면 성능이 떨어지지 않나요?
아닙니다. 주문 데이터가 많아질수록 오히려 패턴 학습이 가능해져 정확도는 점진적으로 개선됩니다.

### Q4. MLOps를 꼭 써야 하나요?
모델을 한 번 쓰고 끝낼 생각이라면 필요 없습니다. 그러나 운영이 계속되는 서비스라면, 재학습·버전 관리·모니터링을 위해 필수적입니다.

## 외부(고객) 관점 FAQ

### Q1. 별도의 앱을 설치해야 하나요?
아닙니다. 기존처럼 문자를 받기만 하면 됩니다. 시스템이 뒤에서 자동으로 처리합니다.

### Q2. 소규모 판매자도 사용할 수 있나요?
네. 하루 몇 건의 주문만 있는 판매자부터 수십·수백 건의 주문을 처리하는 판매자까지 모두 사용할 수 있도록 설계되었습니다.

### Q3. 엑셀 외에 다른 활용도 가능한가요?
가능합니다. 엑셀 외에도 회계, 정산, 배송 시스템과 연동할 수 있는 데이터 구조를 제공합니다.

### Q4. 개인정보나 계좌 정보는 안전한가요?
주문 데이터는 암호화되어 저장되며, 접근 권한은 최소화됩니다. 운영 보안은 시스템 설계의 핵심 요소입니다.
